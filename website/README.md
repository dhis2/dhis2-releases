# DHIS2 Website Content

This folder holds a set of small, self-contained web pages about DHIS2 **apps and core
releases** (the app catalogue, per-app release notes, recent releases and release-timeline
charts). Each page is plain HTML + JavaScript with no build step, served from GitHub Pages
and embedded as iframes on the DHIS2 website.

Pages live in [content/](content/) and read their content at runtime from the JSON files in
[data/](data/), so the pages themselves rarely change — refreshing the content usually means
regenerating the JSON only.

## Data files

These live in [data/](data/) and are the source of truth for the pages above. Several are
generated by the scripts in [tools/](../tools) (for example `app_releases.json` is produced by
`tools/app-release-notes/generate_app_notes.py`).

- [data/apphub.json](data/apphub.json) — Full App Hub catalogue (apps, developers, versions,
  compatibility). Drives the app cards, charts and ReAR views.
- [data/app_releases.json](data/app_releases.json) — Per-app, per-version release notes grouped
  by change type (Features, Bug Fixes, Maintenance). Drives the release-note browser.
- [data/non_cd_apps.json](data/non_cd_apps.json) — Apps not on the continuous-delivery pipeline,
  merged into the app card grid.
- [data/max_apps.json](data/max_apps.json) — Master list of apps with functional category and
  whether each is a core app; used to categorise the cards.

The following are supporting/generated data not currently fetched by the live pages, kept as
inputs or snapshots: [data/apphub_small.json](data/apphub_small.json),
[data/bundled_apps.json](data/bundled_apps.json) and [content/output.json](content/output.json).
